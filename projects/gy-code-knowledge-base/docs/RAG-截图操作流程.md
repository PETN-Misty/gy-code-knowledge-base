## 必做 ④：RAG 流程说明

不需要截图。直接在文档中写入以下 5 句话：

> 1. 用户上传 .txt / .md 文件或通过 API 提交文本，后端按段落和句子自动切成约 500 字的知识块存入数据库。
> 2. 系统从数据库取出所有知识块，提取字符 1~4 元特征，用 TF-IDF 算法计算每个块的稀疏向量，存入内存索引。
> 3. 用户提问时，对问题做同样的向量化处理，与所有知识块向量计算余弦相似度，按得分降序返回最相关的 topK 个块。
> 4. 将命中的知识块拼接成上下文，附上来源标签，传给 DeepSeek 大语言模型。
> 5. DeepSeek 根据提供的参考资料生成带来源引用的中文回答，返回给用户。

## 选做 ⓒ：topK 改变后的影响说明

不需要截图。直接在文档中写入以下说明：

| topK | 命中块数 | 特点 |
|------|---------|------|
| 2 | 2 块 | 回答更聚焦，但可能遗漏辅助信息 |
| 5（默认） | 5 块 | 平衡相关性与上下文丰富度 |
| 8 | 8 块 | 上下文更丰富，但可能引入噪声 |

**影响：**
- topK 越小 → LLM 看到的参考越少，回答更精确但不全面
- topK 越大 → LLM 看到的参考越多，回答可能更全面，但低相关度块会分散注意力、增加 Token 消耗
- 推荐值 3~7，根据知识库规模动态调整
