# 🚀 Spring Boot 项目使用指南（零基础版）

---

## 📖 这是什么项目？

这是一个 **Java Web 项目**，用 **Spring Boot** 框架搭建。  
简单说：它是一个**网站后端程序**，启动后可以在浏览器里访问。

> 本项目已配置好：
> - ✅ 一个测试接口：`http://localhost:8080/hello`（返回 "Hello, Spring Boot!"）
> - ✅ 健康检查：`http://localhost:8080/actuator/health`
> - ✅ 终端日志：每次访问 `/hello` 会在终端打印日志
> - ✅ 国内镜像加速（腾讯云）

## ✅ 第一步：检查环境

### 你需要安装什么？

只需要 **Java 17**（JDK），其他不需要。

**检查是否已安装：**

打开 **命令提示符（CMD）**（按 `Win + R`，输入 `cmd`，回车），然后输入：

```cmd
java -version
```

如果看到类似下面的文字，说明 Java 已装好 ✅：

```
openjdk version "17.0.19" 2026-04-21
Java(TM) SE Runtime Environment ...
```

如果显示 `'java' 不是内部或外部命令`，则需要安装：

## 🔧 常见问题

### Q：启动报错 `java 不是内部或外部命令`
👉 Java 没安装或没配置好，参考上面的安装教程。

### Q：下载 Gradle 很慢或连接失败
👉 已配置腾讯云国内镜像，如果还慢，检查网络连接。

### Q：端口 8080 被占用
👉 修改 `application.properties` 文件中的端口号：
```properties
server.port=8081    ← 改成其他数字
```
然后重新运行。

### Q：`gradlew.bat` 不是内部或外部命令
👉 先 `cd` 进入项目目录：
```cmd
cd C:\Users\li\Desktop\26-6\demo\demo
```
再运行 `gradlew.bat bootRun`

## 💡 我想加新功能怎么办？

### 加一个新页面（接口）

在 `HelloController.java` 中添加：

```java
@GetMapping("/hi")
public String hi() {
    log.info("有人访问了 /hi 接口！");
    return "你好！";
}
```

重新运行，访问 `http://localhost:8080/hi` 就能看到"你好！"，终端也会打印日志。

### 访问时在终端打印日志

`HelloController.java` 中已经配好了日志：
```java
private static final Logger log = LoggerFactory.getLogger(HelloController.class);

@GetMapping("/hello")
public String hello() {
    log.info("有人访问了 /hello 接口！");  // ← 这行会在终端打印
    return "Hello, Spring Boot!";
}
```

### 修改端口号

打开 `application.properties`，修改：
```properties
server.port=9999
```
重启项目，访问 `http://localhost:9999/hello`。

## 🐙 Git 版本管理（已配置好）

本项目已是一个 **Git 仓库**（已提交第一个版本）。

### Git 是什么？

简单说：Git 就像**存档功能**——每次修改代码后可以"存档"一次，方便以后回到任意一个版本。

### 基本操作

#### 查看当前状态
```cmd
git status
```
绿色 = 已存档，红色 = 有修改未存档。

#### 保存修改到 Git（三步走）

改完代码后，想"存档"：

```cmd
:: 第一步：把改过的文件加入暂存区
git add -A

:: 第二步：生成一个版本（存档）
git commit -m "这里写你改了啥"
```

示例：
```cmd
git add -A
git commit -m "添加了 /hi 接口"
```

#### 查看历史版本
```cmd
git log --oneline
```
会显示类似：
```
dfde205 init: Spring Boot demo project
```

#### 回到上一个版本
```cmd
git checkout .
```
**注意**：这会丢弃未存档的修改，谨慎使用。

### 日常开发流程

```
改代码 → git add -A → git commit -m "说明" → 继续改代码 ...
```

每次完成一个小功能就存一次档，养成好习惯 ✅
