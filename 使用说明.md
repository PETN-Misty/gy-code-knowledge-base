# GY · 代码知识库 使用说明书

> 🎯 **面向编程学习者** —— 存储代码、查找代码、通过 AI 学习编程

---

## 📖 这是什么？

这是一个**代码知识库管理工具**，给你一个网页界面，让你可以：

- **存储**各种编程语言的代码片段（JavaScript、Python、Go、Rust 等）
- **浏览**代码，按语言和难度筛选
- **语法高亮**展示代码，方便阅读学习
- **AI 智能搜索**——用中文自然语言查找代码（"找排序算法的例子"）
- **RAG 知识库问答**——上传编程文档，让 AI 基于文档回答问题

项目内置了 **8 个跨语言典型代码示例**，启动即可查看学习。

## 📦 环境要求（安装前准备）

需要提前装好两个东西：

### 1️⃣ Node.js
- 用来运行这个项目
- 检测是否已安装：打开终端（cmd/PowerShell），输入：
  ```bash
  node --version
  ```
- 如果看到版本号（如 `v24.x.x`）说明已安装
- 如果没有，去 https://nodejs.org 下载安装（选 LTS 长期支持版）

### 2️⃣ MySQL
- 用来存储数据
- 本项目使用的是你电脑上已经装好的 MySQL 8.0
- 确认 MySQL 在运行中（桌面右下角有没有 MySQL 图标？）

> ✅ 这两个已经装好了就不用再装

## 🚀 快速开始

### 第一步：打开项目目录

```bash
cd "C:\Users\li\Desktop\26-6\demo2"
```

### 第二步：安装依赖（仅首次）

```bash
npm install
```
- 这步会下载项目需要的一些工具包（依赖）
- 看到 `added xx packages` 就成功了

### 第三步：启动项目

```bash
npm start
```

你会看到这样的画面：

```
====================================================
  🚀  GY·代码知识库 (Code Learning Hub) 已启动
  地址: http://localhost:3000
====================================================
```

> **💡 小技巧**：在 Claude 对话中，可以用 `! npm start` 直接启动，输出会显示在当前对话里。

### 第四步：打开页面

打开浏览器，在地址栏输入：
```
http://localhost:3000
```

> 看到页面就说明启动成功了！**不要关掉那个黑色的终端窗口**，关了项目就停了。

### 第五步：关闭项目

按 `Ctrl + C`（在终端窗口里按），就关闭了。

## 🤖 AI 代码搜索

用中文描述你想找的代码，AI 会自动理解意图并查数据库。

在左侧 **AI 代码搜索** 框里输入问题，比如：

| 你可以问 | AI 会怎么做 |

## 🔌 API 接口（给开发者的）

如果你是开发者，想用程序调用这些接口：

### 基本地址
```
http://localhost:3000
```

### 接口列表

| 方法 | 地址 | 功能 |
|------|------|------|
| `GET` | `/api/tables` | 查看所有表 |
| `GET` | `/api/code_snippets` | 查询代码片段（支持筛选/排序/分页） |
| `GET` | `/api/code_snippets/:id` | 获取单条代码 |
| `POST` | `/api/code_snippets` | 新增代码片段 |
| `PUT` | `/api/code_snippets/:id` | 修改代码片段 |
| `DELETE` | `/api/code_snippets/:id` | 删除代码片段 |
| `GET` | `/api/languages` | 编程语言列表 |
| `POST` | `/api/ai/search` | AI 自然语言搜代码 |
| `POST` | `/api/rag/documents` | 上传文档到知识库 |
| `GET` | `/api/rag/documents` | 查看知识库文档 |
| `DELETE` | `/api/rag/documents/:title` | 删除指定文档 |
| `POST` | `/api/rag/ask` | 基于知识库问答 |

> 通用 CRUD 接口同样适用于任意表，会自动适配表结构。

### 调用示例

**查所有代码**：
```bash
curl http://localhost:3000/api/code_snippets
```

**按语言和难度筛选**：
```bash
curl "http://localhost:3000/api/code_snippets?language=Python&difficulty=beginner"
```

**按标签搜索**：
```bash
curl "http://localhost:3000/api/code_snippets?tags=并发"
```

**新增代码**：
```bash
curl -X POST http://localhost:3000/api/code_snippets \
  -H "Content-Type: application/json" \
  -d '{"title":"Hello World","language":"Python","code":"print(\"Hello World\")","description":"入门示例","difficulty":"beginner","tags":"基础,入门"}'
```

**AI 搜索**：
```bash
curl -X POST http://localhost:3000/api/ai/search \
  -H "Content-Type: application/json" \
  -d '{"query":"找排序算法的代码"}'
```

**RAG 知识库提问**：
```bash
curl -X POST http://localhost:3000/api/rag/ask \
  -H "Content-Type: application/json" \
  -d '{"query":"这个项目怎么用？"}'
```

### 查询参数（筛选/排序/分页）

```
GET /api/code_snippets?language=Python             # 按语言筛选
GET /api/code_snippets?difficulty=beginner          # 按难度筛选
GET /api/code_snippets?tags=排序                     # 按标签搜索
GET /api/code_snippets?_sort=created_at&_order=DESC  # 按创建时间降序
GET /api/code_snippets?_page=1&_size=20              # 分页（第1页，每页20条）
```

### 返回格式

所有接口统一返回 JSON，每条响应都带 `watermark: "GY"`：

```json
{
  "success": true,
  "data": [...],
  "count": 5,
  "watermark": "GY"
}
```

## 🗄️ 数据库表结构

系统在首次启动时自动创建以下表：

### code_snippets（代码片段）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (自增) | 主键 |
| title | VARCHAR(200) | 代码标题 |
| language | VARCHAR(50) | 编程语言 |
| code | TEXT | 代码内容（全文索引） |
| description | TEXT | 说明 |
| tags | VARCHAR(500) | 标签（逗号分隔） |
| difficulty | VARCHAR(20) | 难度: beginner / intermediate / advanced |
| source | VARCHAR(200) | 来源 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### languages（编程语言）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (自增) | 主键 |
| name | VARCHAR(50) | 语言名称（唯一） |
| extension | VARCHAR(20) | 文件扩展名 |
| sort_order | INT | 排序 |

### knowledge_chunks（知识库块）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (自增) | 主键 |
| title | VARCHAR(500) | 文档标题 |
| chunk_text | TEXT | 文本块（全文索引） |
| chunk_index | INT | 块序号 |
| source | VARCHAR(200) | 来源 |
| created_at | DATETIME | 创建时间 |

## ⚠️ 常见问题

### Q: 启动时报错 `port 3000 already in use`
答：端口被占用了。先停掉旧进程再启动：
```bash
taskkill /f /im node.exe
npm start
```

### Q: 打开页面是空白 / 看不到数据
答：先确认终端窗口没关，再确认 `http://localhost:3000` 地址没输错。如果页面显示"暂无代码片段"，等几秒刷新即可（首次启动会自动建表插数据）。

### Q: 页面没有代码数据
答：首次启动会自动插入 8 个示例代码片段（Python冒泡排序、JavaScript Promise、Go goroutine 等），稍等几秒刷新即可。

### Q: AI 搜索返回 "未配置 API Key"
答：检查 `.env` 文件里有没有 `DEEPSEEK_API_KEY`。如果没有，需要申请 DeepSeek API 密钥并填进去。

### Q: 新增/编辑代码后页面没变
答：点击 🔄 刷新按钮刷新一下。

### Q: 如何上传代码文档到知识库？
答：目前支持通过 API 上传：
```bash
curl -X POST http://localhost:3000/api/rag/documents \
  -H "Content-Type: application/json" \
  -d '{"title":"文档标题","content":"文档内容...","source":"来源"}'
```

### Q: 中文显示乱码
答：在终端里看到乱码是正常的（GBK 编码问题），不影响数据。网页上显示正常就行。

### Q: 想重置数据库怎么办？
答：删除 `user_api` 数据库再重新启动即可：
```bash
mysql -u api_user -papi_pass_2024 -e "DROP DATABASE user_api"
npm start
```
