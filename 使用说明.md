# GY · 通用 CRUD API 使用说明书

> 🎯 **面向零基础用户** —— 跟着步骤走，不需要编程经验也能用起来

---

## 📖 这是什么？

这是一个**数据库管理工具**，给你一个网页界面和一个 API 接口，让你可以在浏览器里**增删改查** MySQL 数据库里的数据。

打个比方：它就像 phpMyAdmin 的简化版 + 一个 AI 助手。

---

## 📦 环境要求（安装前准备）

需要提前装好两个东西：

### 1️⃣ Node.js
- 用来运行这个项目
- 检测是否已安装：打开终端（cmd/PowerShell），输入：
  ```bash
  node --version
  ```
- 如果看到版本号（如 `v24.x.x`）说明已安装
- 如果没有，去 https://nodejs.org 下载安装（选 LTS 长期支持版）

### 2️⃣ MySQL
- 用来存储数据
- 本项目使用的是你电脑上已经装好的 MySQL 8.0
- 确认 MySQL 在运行中（桌面右下角有没有 MySQL 图标？）

> ✅ 这两个已经装好了就不用再装

## 🚀 快速开始

### 第一步：打开项目目录

```bash
cd "C:\Users\li\Desktop\26-6\demo2"
```

### 第二步：安装依赖（仅首次）

```bash
npm install
```
- 这步会下载项目需要的一些工具包（依赖）
- 看到 `added xx packages` 就成功了

### 第三步：启动项目

```bash
npm start
```

你会看到这样的画面：

```
====================================================
  🚀  GY·通用 CRUD API (MySQL) 已启动
  地址: http://localhost:3000
====================================================
```

### 第四步：打开页面

打开浏览器，在地址栏输入：
```
http://localhost:3000
```

> 看到页面就说明启动成功了！**不要关掉那个黑色的终端窗口**，关了项目就停了。

### 第五步：关闭项目

按 `Ctrl + C`（在终端窗口里按），就关闭了。

## 🖥️ 网页界面使用指南

打开 `http://localhost:3000` 后：

### 左侧面板

| 区域 | 说明 |
|------|------|
| 📋 **数据表** | 显示数据库里有哪些表，点击切换 |
| 📖 **API 文档** | 接口列表，给开发者看的 |
| 🤖 **AI 智能搜索** | 用中文提问，AI 帮你查数据 |

### 右侧主面板

| 区域 | 说明 |
|------|------|
| 表名标题 | 显示当前查看的是哪张表 |
| 筛选栏 | 按条件筛选数据 |
| 数据表格 | 显示表中的数据，支持编辑和删除 |
| 分页 | 数据太多时分页显示 |
| 响应预览 | 显示原始数据（给技术人员看的） |

### 常用操作

**查看数据**：点击左侧表名 → 右侧显示数据

**筛选数据**：在筛选栏输入条件 → 点击 🔍 筛选

**新增记录**：点击 ➕ 新增 → 填表单 → 确认

**编辑记录**：点击行上的 ✏️ 编辑 → 改数据 → 确认

**删除记录**：点击行上的 🗑️ 删除 → 确认

## 🔌 API 接口（给开发者的）

如果你是开发者，想用程序调用这些接口：

### 基本地址
```
http://localhost:3000
```

### 接口列表

| 方法 | 地址 | 功能 |
|------|------|------|
| `GET` | `/api/tables` | 查看所有表 |
| `GET` | `/api/:table` | 查询表数据 |
| `GET` | `/api/:table/:id` | 获取单条记录 |
| `POST` | `/api/:table` | 新增记录 |
| `PUT` | `/api/:table/:id` | 修改记录 |
| `DELETE` | `/api/:table/:id` | 删除记录 |
| `POST` | `/api/ai/search` | AI 自然语言查询 |

### 调用示例

**查 users 表**：
```bash
curl http://localhost:3000/api/users
```

**新增用户**：
```bash
curl -X POST http://localhost:3000/api/users \
  -H "Content-Type: application/json" \
  -d '{"name":"小明","email":"xm@test.com","age":18}'
```

**AI 查询**：
```bash
curl -X POST http://localhost:3000/api/ai/search \
  -H "Content-Type: application/json" \
  -d '{"query":"30岁以上的用户"}'
```

### 查询参数（筛选/排序/分页）

```
GET /api/users?name=张三             # 按姓名模糊搜索
GET /api/users?age=28                # 按年龄精确搜索
GET /api/users?minAge=20&maxAge=40   # 按年龄范围搜索
GET /api/users?_sort=age&_order=DESC # 按年龄降序排列
GET /api/users?_page=1&_size=10      # 分页（第1页，每页10条）
```

### 返回格式

所有接口统一返回 JSON，每条响应都带 `watermark: "GY"`：

```json
{
  "success": true,
  "data": [...],
  "count": 5,
  "watermark": "GY"
}
```

## ⚠️ 常见问题

### Q: 启动时报错 `port 3000 already in use`
答：端口被占用了。关掉之前的终端窗口，或者重启电脑再试。

### Q: 打开页面是空白 / 看不到数据
答：先确认终端窗口没关，再确认 `http://localhost:3000` 地址没输错。

### Q: AI 搜索返回 "未配置 API Key"
答：检查 `.env` 文件里有没有 API Key。如果没有，找给你这个项目的人要。

### Q: 修改了数据但页面没变
答：点击 🔄 刷新按钮刷新一下。

### Q: 中文显示乱码
答：在终端里看到乱码是正常的，不影响数据。网页上显示正常就行。
